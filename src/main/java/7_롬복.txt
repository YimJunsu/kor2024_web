[ 롬복 ]
    1. 정의 : 반복되는 코드들을 줄여주는 라이브러리 (미리 만들어진 함수/코드)
    2. 내 프로젝트의 롬복 적용/설치 하는 방법
        1) 롬복 관련 build.gradle 에 의존성 추가한다.
            *[ 의존성 검색/추가 : 3.그레이들 메모장 참고 ]
            compileOnly 'org.projectlombok:lombok'
            annotationProcessor 'org.projectlombok:lombok'
        2) 인텔리제이 롬복 설치 ( 인텔리제이[PC] 마다 1번 )
            1] 상단 메뉴 -> [FILE] -> [Setting] -> 플러그인
            2] 롬복 검색/설치

    3. 제공 : @어노테이션 이용하여
        1) 생성자를 대신 만들어준다.
           디폴트 생성자 : @NoArgsConstructor
           모든매개변수 생성자 : @AllArgsConstructor
        2) getter and setter 메소드 대신 만들어진다
            @Getter         : 멤버변수의 getter 메소드
            @Setter         : 멤버변수의 setter 메소드
        3) toString 메소드 대신 만들어 준다.
            @toString       : toString 메소드
        4) 빌더 패턴 지원한다.
            @Builder        : **

    4. 빌더 패턴 : 인스턴스(객체) 만드는 패턴
        1. 정의 : 생성자의 단점을 보완 * new 키워드 없이 인스턴스 만들기
        2. 사용법 : 전제조건은 해당 클래스가 @Builder 이면 가능하다.
            클래스명.builder()
                .멤버변수명1(대입할값)
                .멤버변수명2(대입할값)
                .멤버변수명3(대입할값)
                .build();
        3. 목적 : 인스턴스 생성시 초기화를 조금더 유연하게 하기 위해서 자바의 기본 문법 생성자 대신 빌더 패턴 사용


*생성자란?
- new 뒤에 사용하면서 인스턴스(객체)를 생성할 때 멤버변수 초기화하는 메소드
- 단점이자 생성자 사용의 규칙 :
    1) 상황에 따라 초기화가 다르면 여러 생성자가 필요하다. [오버로딩]
    2) 생성자 관리 힘들다.
    3) 매개변수 순서와 개수를 인수값과 일치해야 한다.
